# Humanoid Control (ROS2 Humble)

## Dependencies

### OCS2 and related libraries

Do not build the whole OCS2 monorepo blindly. Build only required packages.

Example setup:

```bash
git clone https://github.com/chengjie1127/ocs2_ros2.git ~/ocs2_ws/src/ocs2
# plus pinocchio / hpp-fcl as required by OCS2
sudo apt install liburdfdom-dev liboctomap-dev libassimp-dev
```

Build required OCS2 packages (example):

```bash
cd ~/ocs2_ws
colcon build \
  --packages-up-to ocs2_legged_robot_ros ocs2_self_collision_visualization \
  --cmake-args -DCMAKE_BUILD_TYPE=RelWithDebInfo
source install/setup.bash
```

### Mujoco (Python)

```bash
pip3 install mujoco
pip3 install pynput
pip3 install scipy
```

## Build This Repository

```bash
cd ~/humanoid_ws
colcon build --cmake-args -DCMAKE_BUILD_TYPE=RelWithDebInfo --symlink-install
source install/setup.bash
```

## Quick Start

```bash
source install/setup.bash

# Cheat estimator simulation
ros2 launch humanoid_controllers load_cheat_controller.launch.py

# Normal estimator simulation
ros2 launch humanoid_controllers load_normal_controller.launch.py

# NMPC + dummy loop
ros2 launch humanoid_dummy legged_robot_sqp.launch.py
```

## Framework

### Foot Planner

The planner is a switched-contact gait model based on OCS2. Contact mode alternates between left/right support, while swing-foot height is generated by a cubic spline profile.

### State Estimator

A linear Kalman filter estimates trunk position and trunk linear velocity.

State-space form:

$$
\begin{aligned}
\hat{x}_k &= A\hat{x}_{k-1} + Bu_k + \omega_{k-1}, \\
z_k &= H\hat{x}_k + \nu_k.
\end{aligned}
$$

Variables:

$$
\begin{aligned}
x &= [\mathbf{p}_b^T,\ \mathbf{v}_b^T,\ \mathbf{p}_c^T]^T, \\
u &= \mathbf{a}_b, \\
z &= [-\mathbf{p}_{local,c}^T,\ \mathbf{v}_b^T,\ \mathbf{h}_c^T]^T.
\end{aligned}
$$

Where:

- $b$: trunk (base)
- $c$: contact point
- $local$: trunk-local frame

Only stance-foot kinematics are trusted strongly. Swing-foot measurement channels are assigned high noise (low trust), which improves robustness during contact switching.

### NMPC

At each cycle, NMPC solves:

$$
\begin{aligned}
\min_{\mathbf{u}(\cdot)}\quad & \phi\big(\mathbf{x}(t_f)\big) + \int_{t_0}^{t_f} l\big(\mathbf{x}(t),\mathbf{u}(t),t\big)\,dt \\
\text{s.t.}\quad & \dot{\mathbf{x}}(t)=\mathbf{f}\big(\mathbf{x}(t),\mathbf{u}(t),t\big), \\
& \mathbf{g}_1\big(\mathbf{x}(t),\mathbf{u}(t),t\big)=\mathbf{0}, \\
& \mathbf{g}_2\big(\mathbf{x}(t),t\big)=\mathbf{0}, \\
& \mathbf{h}\big(\mathbf{x}(t),\mathbf{u}(t),t\big)\ge 0.
\end{aligned}
$$

Centroidal-model state and input are:

$$
\mathbf{x}=[\mathbf{h}_{com}^T,\mathbf{q}_b^T,\mathbf{q}_j^T]^T,\qquad
\mathbf{u}=[\mathbf{f}_c^T,\mathbf{v}_j^T]^T.
$$

In this project, we added and used soft constraints (relaxed barrier) in NMPC, including:

- Friction cone soft constraint
- Joint soft-limit constraint
- Self-collision soft constraint

Other key constraints include zero swing-foot force, stance-foot no-slip style constraints, swing-foot trajectory constraints, and foot roll constraints.

The transcription uses multiple shooting, and the resulting NLP is solved with SQP/IPM workflows (HPIPM backend for QP stage).

### WBC

WBC solves a QP at the current control instant:

$$
\begin{aligned}
\min_x\quad & \tfrac{1}{2}x^THx + x^Tg, \\
\text{s.t.}\quad & lb_A \le Ax \le ub_A, \\
& lb \le x \le ub.
\end{aligned}
$$

Optimization variable:

$$
x=[\mathbf{a}_b^T,\mathbf{a}_j^T,\mathbf{f}_c^T,\mathbf{\tau}_j^T]^T.
$$

Typical task split:

- Cost tasks: base accel/orientation tracking, swing-leg tracking, contact-force regularization
- Constraint tasks: floating-base EOM, torque limits, friction constraints

### PD Controller

MRT provides optimized state/input trajectories. WBC provides accelerations and torques. Final desired commands are generated by:

$$
\begin{aligned}
\mathbf{q}_{des} &= \mathbf{q}_{opt} + \tfrac{1}{2}\mathbf{a}_j t^2, \\
\mathbf{v}_{des} &= \mathbf{v}_{opt} + \mathbf{a}_j t, \\
\mathbf{\tau}_{des} &= \mathbf{\tau}_j.
\end{aligned}
$$

### Typical Frequencies

| Module | Frequency |
|---|---|
| NMPC | 100 Hz |
| MRT | 500 Hz |
| WBC | 500 Hz |
| PD Controller | > 1000 Hz |
| State Estimation | 500 Hz |
